Before coding anything, a well thoughtout design must be in place. This reduces the time spent adjusting code and leads to overall better formatting. While the end result may diverge from the original design, the basic design and structure is the stepping stone needed before beginning to code. See the intial design below.

img

With the design in place, next was deciding how I wanted to maintain the file system. Since this is a follow up project of my trial-run webpage: WhatAKerfuffle.ml, I set out to improve upon the mistakes previously made. The most pressing being an organized file structure. While the original webpage wasn’t terrible, it was a very manual system with lots of refactoring. This time, I sought out for a static webpage file management system. Luckily, I stumbled upon a fantastic YouTube Channel called Codecourse. He introduced me to Jekyll.



"Jekyll is a simple, blog-aware, static site generator perfect for personal, project, or organization sites. Think of it like a file-based CMS, without all the complexity. Jekyll takes your content, renders Markdown and Liquid templates, and spits out a complete, static website ready to be served by Apache, Nginx or another web server. Jekyll is the engine behind GitHub Pages, which you can use to host sites right from your GitHub repositories"


It took a couple of days of hammering through examples and trial-and-error processes, but by the end I had a comprehensive understanding of the file system, variables, imports, and site generation.

img of basic file system

After tackling this system, it was a matter of programming the back-end and ensuring the layout was adaptive and all-browsers compatable.

Mulitple small adjustments were made to the design since starting the project. Most of these adjustments were suggestions from one of my professors, Dr. Armstrong. Her feedback was invaluable and resulted in multiple small adjustments of the overall project.

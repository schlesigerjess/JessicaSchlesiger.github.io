First thing's first: host the site somewhere and find a domain for free, if possible. This part arguable took the longest out of the whole project, but ultimately I did find a solution! After scouring the internet, I stumbled on Github Pages. Github Pages allows users to create repositories that host webpages. This will result in a URL along the lines of: http://username.github.io/repository, but they do support custom URLs at the cost of not being https secure. Since this webpage does not collect any user information and the site's content is completely open source, not being https secure is an acceptable cost of having a custom domain.

This is where Freenom.com comes in. Freenom allows users to claim domains for free - although they'll be a domain with an alternative protocol identifier like .ml and .tk. While that's not ideal, it is still a free domain. I chose the title What A Kerfuffle as kerfuffle is my favorite word, and I use it for most of my personal accounts. This resulted in the webpage's URL: WhatAKerfuffle.ml.
